Тестовое задание: Карусели “Генерация + Редактор”
Референс: Draft https://getdraft.io/

Контекст
У нас есть функционал, который генерирует карусели для Instagram из разных источников (текст/ссылки/видео). Нужно собрать мини-продукт (MVP), который закрывает end-to-end сценарий: создание, настройка формата, редактор, экспорт.
Главное сделать готовый MVP флоу на беке и фронте рабочий end-to-end.

Цель
Собрать MVP, где пользователь может:
создать карусель из исходных данных (текст/видео)
настроить формат (кол-во слайдов, язык, стиль текста)
запустить генерацию (LLM) → получить структуру слайдов
отредактировать карусель в простом редакторе (шаблон/фон/макет/шапка/подвал)
экспортировать в набор изображений (PNG/JPG)
интерфейс десктоп + мобильный
Дизайн в фигме: https://www.figma.com/design/rg1ZgIv8fDeyGAeFe71Z9u/%D0%A8%D0%B0%D0%B1%D0%BB%D0%BE%D0%BD%D1%8B-%D0%BA%D0%B0%D1%80%D1%83%D1%81%D0%B5%D0%BB%D1%8C--Copy-?node-id=0-1&t=b6c1fWlCzwWTegcr-1

Что обязательно сделать
1) UI: десктоп + мобильнная
Можно любым способом:
минимальный фронт Nuxt (vue)
Критерий: список каруселей, создание, генерация и редактор должны быть удобно кликабельны.

2) Список каруселей (My carousels)
Нужно:
список/грид каруселей
карточка: title, created_at, превью (первый слайд), status (draft/generating/ready/failed)
метки (опционально): язык, кол-во слайдов
CTA: Создать, Открыть, Продолжить в редакторе

3) Создание карусели (выбор способа)
Минимум 2 способа (можно 3):
Из текста: пользователь вставляет текст поста/черновик
Из видео: загрузка видео или ссылка
(опционально) Из ссылок/материалов: список ссылок + заметки (bullet points)

После выбора способа форма “Изменить формат”:
Кол-во слайдов: 6–10
Язык: RU / EN / FR (минимум RU+EN достаточно)
Стиль текста: поле “пример поста” (пользователь вставляет кусок текста), из него делаете style_hint для LLM

4) Генерация (LLM)
Нужно:
кнопка “Сгенерировать” + отображение “спишется N токенов”
статусы: queued → running → done/failed
результат: массив слайдов (например 8 шт.), у каждого:
order (1..N)
title (короткий заголовок)
body (основной текст)
footer/cta (опционально)
Требование: генерация асинхронная, чтобы UI не зависал.
Рекомендация по промпту:
вход: source_text + links + notes + language + slides_count + style_hint
выход: JSON со слайдами (валидация схемой)
ограничения по длине (чтобы текст помещался в карточку)

5) Редактор карусели (минимально, но рабочий)
Нужно:
переключение между слайдами
редактирование текста на слайде (title/body)
панель настроек (табами или модалками):
Шаблон: выбор из 2–3 пресетов (например Classic / Bold / Minimal)
Фон: цвет или фото (upload) + опция “затемнение” (toggle или слайдер)
Макет: отступы контента (слайдер) + выравнивание (гор/верт)
Дополнительно: показывать шапку/подвал (toggles) + поля “текст шапки/подвала”
опция “Применить ко всем слайдам” (можно хотя бы для фона/шаблона)

6) Экспорт
Минимум:
кнопка “Экспорт” скачать ZIP с N картинками (slide_01.png …)
размер слайда фиксированный: 1080×1350 (4:5)
рендер любым способом:
серверный HTML изображение (playwright/puppeteer)
canvas на фронте
SVG в PNG
Если экспорт сложно допускается упрощение:
отдавать HTML-превью и “скриншотить” только 1–2 слайда
НО: в README четко описать ограничения.

Технические требования
Python + FastAPI
нормальная структура проекта
БД: Postgres 
хранение файлов: S3/MinIO
асинхронность:
BackgroundTasks
Docker-compose (очень желательно)

Минимальный API (пример)
Карусели
GET /carousels?status=ready&lang=ru
GET /carousels/{id}
POST /carousels (создать черновик: title, source_type, source_payload, format)
PATCH /carousels/{id} (обновление title/format)
Генерации
POST /generations (запуск генерации + списание токенов)
GET /generations/{id} (статус + результат)
Слайды
GET /carousels/{id}/slides
PATCH /carousels/{id}/slides/{slide_id} (редактирование текста)
Настройки дизайна
PATCH /carousels/{id}/design (template/bg/layout/header/footer)
Файлы
POST /assets/upload (multipart) фото фона/ресурсы
Экспорт
POST /exports (запуск экспорта)
GET /exports/{id} (статус + ссылка на ZIP)

Плюсом будет (не обязательно)
WebSocket/SSE для статусов генерации/экспорта
история генераций по карусели (re-generate)
простая авторизация (API-key/токен)
валидация JSON-результата (Pydantic schema) + ретраи
логи/трейсинг
